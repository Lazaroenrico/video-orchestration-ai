Como nasce a imagem

  • assign_voice_profile resolve o perfil do creator antes da imagem.
  • openai_image.py monta um prompt seguro de retrato comercial.
  • O genero do VoiceProfile injeta uma clausula brand-safe: adulto homem ou adulta mulher.
  • GPT Image 2 retorna URL ou b64_json; o adapter normaliza para URI.
  • O upscale roda no Replicate Real-ESRGAN e a midia e persistida localmente.

Como nasce a voz

  • ReplicateVoiceAdapter chama ElevenLabs hospedado no Replicate.
  • O texto do TTS usa creator voice {index} mais preset e briefing humano.
  • Os campos do modelo sao configuraveis por env: texto, voz, model id e JSON base.
  • Vozes podem ser pools CSV por genero; o indice escolhe de forma deterministica.
  • Reroll avanca no pool do mesmo genero, preservando a imagem.

Padroes que governam tudo

  • Ports/Protocols: cada provider implementa contratos em adapters/base.py.
  • CompositeAdapter: o grafo fala com um objeto, mas cada papel roteia para um adapter.
  • Config-driven: YAML troca providers sem alterar a topologia.
  • Determinismo: mock sem random, ids por hash e testes offline.
  • Human-in-the-loop: approval pausa o grafo e retoma via Command(resume=...).
  • Rate-limit consciente: retry trata 429 e throttle global espaça chamadas Replicate.

Regra pratica: se voce esta mudando provider, mexa em adapter/registry/config. Se esta mudando fluxo, mexa no grafo/nodes. Se esta mudando operacao da UI, mexa no server web e no contrato SSE.
